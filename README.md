# ReadLog

ReadLog is a responsive reading tracker built with Next.js App Router, TypeScript, Tailwind, shadcn-style UI, TipTap, and Supabase.

## What changed
- Cross-device sync via Supabase Auth + Postgres.
- Protected routes (`/login`, `/signup` public; app routes require login).
- Cover uploads stored in Supabase Storage.
- Add/Edit Book modal UX fix:
  - fixed-size cover preview
  - reserved preview area (no layout jump)
  - remove image action
  - scrollable modal body (`max-h-[80vh]`)
  - sticky action footer so submit is always reachable

## Local development
1. Install dependencies:
```bash
npm install
```
2. Copy env template and set values:
```bash
cp .env.example .env.local
```
3. Run dev server:
```bash
npm run dev
```
4. Open: `http://localhost:3000`

## Required environment variables
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

Env validation is enforced in `src/lib/supabase/env.ts`.

## Supabase setup
1. Create a Supabase project.
2. Open SQL Editor and run:
- `supabase/schema.sql`
3. In Authentication:
- Enable Email/Password provider.
4. Storage:
- The SQL creates bucket `readlog-covers` + policies.

## Database schema (high level)
Main tables:
- `books`
- `chapter_notes`
- `book_notes`
- `vocab_entries`

Additional synced tables used by the app:
- `app_settings`
- `activity_log_days`

All tables use RLS with `user_id = auth.uid()` ownership policies.

## Repository layer
Supabase CRUD is organized under:
- `src/lib/repositories/book-repo.ts`
- `src/lib/repositories/chapter-repo.ts`
- `src/lib/repositories/note-repo.ts`
- `src/lib/repositories/vocab-repo.ts`
- `src/lib/repositories/reading-data-repo.ts`

## Auth and route protection
- Auth context: `src/lib/context/auth-context.tsx`
- Middleware guard: `middleware.ts`
- Auth pages:
  - `/login`
  - `/signup`
- Header user menu includes logout.

## Backup / restore
In `Settings`:
- `Export Backup` downloads a JSON snapshot.
- `Import Backup` replaces current synced data after confirmation.

## Build, test, lint
```bash
npm run lint
npm run test
npm run build

```

## Vercel deployment
1. Push the project to GitHub.
2. Import the repo in Vercel.
3. Set environment variables in Vercel Project Settings:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. Deploy.

Default commands:
- Build: `next build`
- Start: `next start`

## SEO / metadata
- Metadata set in `src/app/layout.tsx`
- `robots.txt` generated by `src/app/robots.ts`

## Notes
- The app is fully functional offline for UI interactions, but canonical persistence is Supabase sync.
- Storage upload paths are user-scoped: `${user_id}/...` in `readlog-covers`.
